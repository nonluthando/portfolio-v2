How I work

A practical engineering approach.

The goal is not only to produce working code, but to make the behaviour understandable, verifiable and maintainable.

01

Define behaviour clearly

I prefer explicit domain rules, useful acceptance criteria and outputs that can be explained.

02

Protect important logic

Validation, authorization and automated tests are part of implementation rather than later cleanup.

03

Use AI as an engineering tool

I use AI for architecture critique, planning, test ideas, debugging hypotheses and documentation, while retaining responsibility for design decisions, code review, behaviour tracing and final verification.

04

Keep product context visible

Technical choices should support the user workflow and the actual risk or friction the product addresses.
