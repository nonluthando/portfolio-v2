Live Project

ScoreRent

An explainable rental decision-support platform for renters deciding where to spend non-refundable application fees.

ScoreRent compares a renter profile with listing requirements and returns a confidence score, verdict, reasons, recommended actions and a transparent scoring breakdown.

Open Live App

Implemented Stack

Backend: Python and FastAPI

Data: PostgreSQL

Quality: pytest and GitHub Actions

Delivery: Render

Why I Built It

The idea came from a problem I had seen around me.

I built ScoreRent after seeing people move from university residence into private rentals. Applicants were paying fees without knowing whether affordability, missing documents, guarantor requirements, bursary coverage or demand made an application unrealistic.

I wanted to make that decision clearer before money was spent. The result shows what strengthens or weakens an application and what the renter can realistically improve.

Implemented Product Flows

These features are available in the deployed application.

Explainable Evaluation

Guest and authenticated users receive a score, verdict, confidence level, reasons, recommended actions and a rule-by-rule scoring breakdown.

Authentication and Profiles

Users can sign up, sign in, save renter information and reuse profile data across evaluations.

Evaluation History

PostgreSQL stores users, profiles and past evaluations so results can be revisited later.

Different Renter Contexts

The rules account for workers, students, bursary support, guarantors, affordability, required documents, deposits and listing demand.

Engineering Decisions

The score is useful because its behaviour can be understood and tested.

Deterministic Scoring

The evaluator uses explicit rules. The same inputs produce the same result, and every score adjustment can be traced.

Confidence, Not Probability

I call the result a confidence score because the available information does not support a reliable acceptance probability.

Automated Tests

pytest coverage protects behaviour across affordability, documents, student support, guarantor logic, verdicts, reasons and recommended actions.

CI/CD and Deployment

GitHub Actions runs the test suite on repository changes. The application is deployed on Render with PostgreSQL persistence.

Planned Evolution

Future work will build on the existing decision-support logic.

Grounded Explanations

A future assistant could explain results in conversational language and answer rental questions using trusted policy and guidance sources.

Dynamic Re-evaluation

Saved listings could be re-evaluated when a user changes income, documents, bursary information, guarantor details or other profile evidence.

Change Alerts

Users could be notified when a profile change makes a saved application meaningfully stronger or weaker.

Current-Profile Comparisons

Listing comparisons could use the renter's latest profile rather than an older snapshot.

What I Learned

ScoreRent strengthened how I think about domain rules, system boundaries, regression testing, database-backed user flows and keeping behaviour predictable while a live project changes.

It also reinforced something I care about in decision-support software: users should be able to understand a result instead of being asked to trust a number without context.
