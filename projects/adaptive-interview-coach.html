Live Product

ViewCoach

A structured interview-preparation platform for creating, importing and practising technical, behavioural and repository-debugging material.

ViewCoach is designed around a larger product question: “Given the interview I have coming up, what should I practise now?” The live version establishes the production foundation and question library needed before planning and AI features are added.

Open Live AppView Repository

Implemented Stack

Core: Django 5.2 modular monolith

Data: PostgreSQL in local, test and production environments

Quality: pytest, Ruff and GitHub Actions

Deployment: Render

The Product Decision

Most interview-preparation tools help users store material. ViewCoach starts from a different problem: preparation already requires a lot of decisions, and choosing what to study each day adds more friction before the work begins.

The current live version focuses on structured material and reliable workflows. Spaced repetition, review scheduling, the Focus Engine, goals and AI are intentionally not claimed as implemented yet.

What Is Live

The deployed product currently contains Milestones 0, 1 and 1.1.

Production Foundation

Custom email-based users, registration, login, logout, protected dashboard, PostgreSQL environments, a database-aware health endpoint, CI and production settings.

Structured Question Library

Technical, behavioural and repository-debugging question types with type-specific create, edit and detail experiences.

Private Study Workflows

User-owned libraries, authorization rules, search and filters, progressive technical reveals, debugging diagnosis reveals and dashboard summaries.

Bulk Import and Review

Imports from pasted text, TXT, Markdown, CSV, DOCX and text-based PDFs, with resumable review batches, duplicate detection, editable titles, type overrides and transactional creation.

Architecture Decisions

The architecture is intentionally simple enough to understand and strong enough to grow.

Modular Monolith

Django apps provide domain boundaries without introducing distributed-system complexity the project does not need.

PostgreSQL From the Start

Local, test and production environments use PostgreSQL so development behaviour matches production more closely.

Typed Question Models

A concrete base question with typed child models allows each interview format to store the information it actually needs.

Deterministic Before AI

Structured product logic remains separate from future AI behaviour so planning and progress can stay inspectable and testable.

Next Milestones

These are future features, not part of the current live claim.

Practice and Review History

Practice records, self-assessment, mistakes, confidence and review history.

Focus Engine

Deterministic recommendations based on interview context, curriculum coverage, weak areas, available time and previous reviews.

Goals and Scheduling

Interview goals, target dates, availability and readiness evidence.

Grounded AI Support

Later retrieval and explanation features with citations, evaluation and safe fallbacks.
