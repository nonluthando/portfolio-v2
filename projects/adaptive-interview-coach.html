Product overview

Structured preparation before automated planning.

ViewCoach starts with the underlying study material and workflows. Users can maintain a private question library, work with different interview formats, progressively reveal answers and import existing material for review.

Spaced repetition, review scheduling, the Focus Engine, goals and generative AI are intentionally separated from the current release. The structured foundation is designed to remain useful and testable before those layers are added.

Core product question

Given the interview context, what should the user practise next?

Architecture

Decisions made for clarity and controlled growth.

Modular Django monolith

Clear domain boundaries without introducing distributed-system complexity before the product requires it.

PostgreSQL from the beginning

One relational data model across local, test and production environments, with no application SQLite fallback.

Concrete base question with typed children

Shared behaviour remains central while each interview format stores and presents its own relevant detail.

Deterministic logic before AI

Study planning and review behaviour will remain visible, explainable and testable independently of future AI features.

Development workflow

AI-assisted, verification-led delivery.

I use AI to challenge architecture choices, turn agreed decisions into implementation plans, generate test ideas, investigate bugs and improve documentation. I retain ownership of the product rules, architecture, trade-offs and acceptance criteria, then review the code, trace the behaviour and run the tests before accepting a change.
