ScoreRent

An explainable rental decision-support application that evaluates renter and listing information before the user commits to a potentially non-refundable application fee.

Product focus
Explainability, confidence and actionable outcomes

Problem and approach

Reduce uncertainty without pretending to predict acceptance.

Rental applications can require fees and substantial documentation before a renter knows whether the listing is realistically suitable. ScoreRent makes the underlying factors visible before that commitment.

The application compares listing requirements with the renter profile and applies deterministic scoring rules. The result includes a score, verdict, confidence level, contributing reasons and practical actions. The product presents this as decision support rather than a guarantee.

Design principle

The user should be able to understand why the result changed.

Engineering decisions

Explainability is part of the architecture.

Deterministic scoring

Important thresholds and penalties remain explicit, inspectable and directly testable.

Structured output

The score is accompanied by reasons and actions so that the result can support a decision rather than replace it.

Tests around domain behaviour

Changes to affordability, documentation, guarantor and bursary rules are protected by automated tests.

Incremental product growth

Authentication, persistence and deployment were added around the core evaluation behaviour without hiding the rules.
