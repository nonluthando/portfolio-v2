ScoreRent

Backend · Live

ScoreRent is a live, iteratively developed backend web application that helps renters evaluate property listings before paying application fees.

Live demo Repository Back to Projects

Overview

Renters often pay non refundable application fees with limited information about whether a listing is worth pursuing. ScoreRent provides a structured, criteria based way to evaluate listings, helping users make informed decisions before committing money.

My Role

I designed and implemented the backend system end to end, including API design, scoring logic, input validation, persistence, automated testing, and deployment. The system was built to support ongoing iteration rather than a one off implementation.

Key Features

  • Criteria based scoring logic for structured listing evaluation
  • Typed input validation using Pydantic models
  • REST API structure with clear endpoints and predictable responses
  • Saved evaluations with persistent storage
  • Live deployment on Render for real world testing and iteration
Scoring Engine Saved evaluations Validation Live deployment

Testing and Quality

I implemented automated tests for the core evaluate flow using pytest, focusing on correctness of the scoring logic and input handling. I also set up CI with GitHub Actions to run tests on every push, helping keep the API stable as I iterate and add new features.
