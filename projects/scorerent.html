ScoreRent

Live · Backend

A rental decision-support web app that helps renters evaluate whether a listing is worth applying for before paying non-refundable application fees.

Live demo Repository Download CV

What ScoreRent does

Renters often pay application fees without enough information about whether the listing is realistic for their financial situation or documentation. ScoreRent provides an explainable evaluation workflow that generates a confidence score (0–100) and a verdict: Worth applying, Borderline, or Not worth it.

The score is based on affordability, document-fit, and demand weighting. It is intentionally designed as a confidence score (not acceptance probability).

Key product flows (what’s actually built)

Guest evaluation flow

Users can evaluate a listing without signing up by entering renter details and listing requirements. The system returns a score, verdict, and clear reasons plus recommended actions.

EvaluateExplainable outputRecommended actions

Logged-in user flow

Users can create an account and log in to save a renter profile and keep an evaluation history. This turns ScoreRent into an ongoing tool rather than a one-off calculator.

Sign upLoginSessionsProfileHistory

Dashboard + saved evaluations

Logged-in users get a dashboard view of their most recent evaluation and can review saved listings they have evaluated previously. Evaluations are persisted and grouped per user.

DashboardSaved evaluationsPersistence

Explainable scoring breakdown

ScoreRent shows how the score was calculated by displaying rule-by-rule adjustments (before → after), including affordability deductions, missing documents, demand pressure, and verdict assignment.

My role / what I built

ScoreRent is an independently built project that I designed and implemented end-to-end. I built the backend logic, the scoring engine, persistence, authentication flow, and deployment.

Scoring engine

I designed a deterministic scoring engine that outputs a confidence score (0–100) and provides explainable reasons and actions. This allows users to understand what lowered the score and what can be improved.

Deterministic scoring Explainable verdicts Rule-based evaluation

Core rules implemented

The evaluator includes affordability thresholds, document-fit matching, and demand weighting. It supports realistic renter scenarios (including student logic).

25/30/35% affordability bands Document-fit rules Demand weighting Student rules
