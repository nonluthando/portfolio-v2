ScoreRent

Rental decision-support web app that helps renters evaluate listings before paying non-refundable application fees.

Problem

Prospective renters can lose money repeatedly on application fees when they apply for listings that are unlikely to fit their affordability, documentation or renter-profile constraints. The problem is not only rejection. It is unclear rejection, where the renter does not know what weakened the application.

What I built

ScoreRent compares a renter profile with listing requirements and returns a 0–100 confidence score, a verdict, reasons, recommended actions and a score breakdown. It supports guest evaluations as well as authenticated flows with saved profiles and evaluation history.

My contribution

  • Designed and implemented the rule-based evaluation engine.
  • Built FastAPI routes, Jinja2 templates and PostgreSQL persistence.
  • Implemented authentication, saved renter profiles and evaluation history.
  • Added pytest tests for scoring behaviour and GitHub Actions for CI.
  • Deployed the app to Render using PostgreSQL persistence.

Technical implementation

The evaluator starts from a baseline score and applies deterministic adjustments based on affordability, documents, renter type, bursary support, guarantor availability and demand level. FastAPI handles routing and validation. PostgreSQL stores users, profiles and evaluations. Jinja2 templates provide the web interface and session-based authentication controls logged-in flows.

Engineering decisions

  • Used rules instead of machine learning because the output needed to be explainable and trustworthy.
  • Kept score breakdowns visible so users can understand why a result changed.
  • Supported guest evaluations to reduce friction before sign-up.
  • Used automated tests to protect business logic from regressions.

Testing and validation

I wrote unit tests for the evaluator logic using pytest and configured GitHub Actions to run tests on push and pull request. I also manually validated sign-up, login, profile creation, evaluation saving and dashboard history flows.

What I learned

ScoreRent taught me how to build a project where technical correctness and user trust matter at the same time. I learned to keep system boundaries clear, test business logic carefully and make decision-support outputs understandable rather than opaque.

Future improvements

Future improvements include refining scoring rules from feedback, strengthening validation, adding clearer onboarding, improving UI polish and expanding external data inputs where they would genuinely improve decision support.
