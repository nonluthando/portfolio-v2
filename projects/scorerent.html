ScoreRent

A rental decision-support app that helps renters judge their fit for a listing before paying non-refundable application fees.

The product evaluates a renter profile against listing requirements and returns a confidence score, a verdict, reasons behind the decision, and practical actions to improve the application.

Stack FastAPI, Jinja2, PostgreSQL, pytest, GitHub Actions, Render
Focus Decision support, product logic, testing, persistence, and deployment
Status Live and working, with future product expansion planned
Open live app Back to projects

The problem

I built ScoreRent after seeing how expensive and uncertain rental applications can be. People were paying application fees, submitting documents, and then getting rejected for reasons they could have known upfront.

I wanted a tool that could make the situation clearer. Not a vague recommendation. A structured result with visible logic and useful next steps.

The shape of the solution

The app compares listing requirements with a renter profile and runs rule-based scoring. The output is intentionally explainable. It is a confidence score, not a fake promise of acceptance.

Core product features

Guest evaluation flow

Users can enter listing details and renter information without signing in and immediately receive a score, verdict, reasons, and suggested actions.

Authentication and saved history

Logged-in users can save renter details, keep evaluation history, and revisit past results.

Explainable scoring

The scoring engine applies deterministic rules and returns a clear breakdown of what changed the score.

Engineering discipline

The project includes tests for the scoring logic, CI checks, and a deployed environment with PostgreSQL persistence.

What I learned

Product lesson

Trust matters in a decision-support product. The user needs to understand the result and feel that the logic is fair, not magical.

Engineering lesson

The more important the logic is, the more it needs tests. This project made me much more disciplined about protecting behaviour as a product grows.

Next steps

These are product improvements I would like to build next.

Dynamic re-evaluation Update earlier results when a renter profile changes.
Smarter comparisons Compare multiple listings using the renter's current profile.
