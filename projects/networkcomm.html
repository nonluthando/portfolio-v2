Overview

A practical networking project focused on real-time communication, concurrent execution, and client-server coordination.

NetworkComm is a Python-based TCP client-server messaging application built to explore networking, concurrency, and protocol design through real-time communication.

The system allows multiple clients to connect to a central server and exchange messages using TCP sockets, multithreading, and a text-based messaging protocol.

The project prioritised correctness, coordination, and realistic networking behaviour over interface complexity while providing practical exposure to client-server architectures and concurrent execution.

Architecture

Client-server coordination and concurrent execution model.

The application follows a centralised client-server architecture. The server listens on a configurable host and port while clients establish TCP connections and register identities.

For every connected client, the server creates a dedicated handler thread responsible for receiving messages, routing communication, maintaining state, and managing disconnect behaviour.

Messages are routed according to communication type, including broadcast messaging, private communication, and room-based interactions.

Client-side sending and receiving are handled concurrently using separate threads to preserve responsiveness and avoid blocking socket reads.
