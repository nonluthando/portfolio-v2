NetworkComm

Problem

Networking and concurrency are easier to understand when they are built directly. I wanted a project that made client-server coordination, socket communication, shared state and failure handling visible.

What I built

NetworkComm is a Python TCP messaging system where multiple clients connect to a central server. It supports broadcast messages, private messages, chat rooms, visibility controls, retry behaviour and graceful disconnect handling.

My contribution

  • Extended and refined the server by improving reliability, readability and coordination behaviour.
  • Implemented the client independently, including user interaction and connection handling.
  • Added retry logic and concurrent send/receive behaviour.
  • Improved message formatting, routing and disconnect handling.
  • Explored failure scenarios and state consistency across connected clients.

Technical implementation

The server listens on a configurable host and port. Each connected client gets a dedicated handler thread responsible for receiving messages, routing them and managing disconnect behaviour. The client uses separate threads for sending and receiving so incoming messages do not block user input.

Engineering decisions

  • Used a centralised server model to keep coordination understandable.
  • Used a thread-per-client approach to practise concurrency directly.
  • Kept the protocol text-based to make messages easy to inspect while debugging.
  • Intentionally kept authentication, encryption and persistence out of scope to focus on networking fundamentals.

Testing and validation

I tested the system manually by running multiple clients at the same time, checking broadcast, private messaging, room flows, disconnects and retries. Logging helped trace message routing and state updates.

What I learned

NetworkComm strengthened my understanding of TCP/IP communication, concurrency, message ordering, retry behaviour and failure handling. It made networking concepts feel concrete rather than abstract.

Future improvements

Future enhancements include message framing, authentication, TLS, persistent chat history, asynchronous I/O, automated tests and performance benchmarking with larger client loads.
